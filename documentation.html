üéØ Project Overview

Samawa Wedding Template is a modern, responsive, and easily customizable wedding website template. This template uses modern web technologies with elegant and user-friendly design.

üí° Important Information: This is a UI-only template designed specifically for wedding organizer businesses. It includes booking forms, galleries, testimonials, and package detail pages, but no backend functionality - it's purely frontend/static.

Technologies Used:

  • HTML5 - Web page structure
  • CSS3 - Styling and animations
  • JavaScript (Vanilla) - Frontend interactivity
  • Tailwind CSS v4 - Utility-first CSS framework
  • Swiper.js - Carousel/slider component
  • MCDatepicker - Date picker component

‚öôÔ∏è Setup & Installation

System Requirements:

  • Node.js (version 16 or newer)
  • NPM or Yarn
  • Text Editor (VS Code recommended)
  • Modern web browser

Installation Steps:

  1. Clone or Download Project
    Download all project files to your local folder.
  2. Install Dependencies
    Open terminal/command prompt in the project folder and run: 
    npm install
  3. Run Development Server
    To start development mode with auto-reload: 
    npm run dev
  4. Build for Production
    To create compiled CSS files: 
    npm run format
‚úÖ Setup Complete! Your website is ready to use. Open the file src/pages/index.html in your browser to see the result.

üé® Customization Guide

1. Changing Brand Colors

The template uses pink (#ff48b6) as the primary color. To change it: 

 /* In src/css/input.css file */
 @theme {
   --color-pink: #your-color-here;  /* Replace with your desired color */
   --color-brown: #5c2d2d;
   --color-green: #22c55e;
   --color-gray: #f7f7f9;
   --color-red: #07b704;
 }

2. Changing Content

Edit HTML files in the src/pages/ folder to change:

  • Text and content
  • Images (replace in public/images/ folder)
  • Contact information
  • Package prices

3. Adding New Pages

  1. Create a new HTML file in src/pages/
  2. Copy structure from existing pages
  3. Update navigation in navbar.js file
  4. Update paths in change-path.js if needed

4. Changing Logo

Replace the public/icons/logo.svg file with your logo. Make sure to use SVG format for best quality.

‚ö†Ô∏è Note: After changing CSS files, run npm run dev to see the changes.

üöÄ Deployment

1. Build for Production

 # Compile CSS
 npm run dev

 # Format code
 npm run format

2. Upload to Web Server

Upload all files to your web server:

  • Upload src/pages/ folder as website root
  • Upload public/ folder for assets
  • Make sure output.css is compiled

3. Web Server Configuration

 # Apache .htaccess
 RewriteEngine On
 RewriteCond %{REQUEST_FILENAME} !-f
 RewriteCond %{REQUEST_FILENAME} !-d
 RewriteRule ^(.*)$ index.html [QSA,L]

 # Nginx
 location / {
     try_files $uri $uri/ /index.html;
 }

4. Performance Optimization

  • Compress images before upload
  • Enable GZIP compression on server
  • Set cache headers for static assets
  • Minify CSS and JavaScript for production
üí° Tips: Use CDN for static assets like images and icons for faster loading.

üîß Troubleshooting

Common Issues & Solutions:

1. CSS not loading

Problem: Styling not appearing or broken
Solution:
  • Make sure to run npm run dev
  • Check CSS file paths in HTML
  • Clear browser cache

2. JavaScript not working

Problem: Modal, datepicker, or other features not working
Solution:
  • Check browser console for errors
  • Make sure all script files are loaded
  • Verify CDN links (Swiper, MCDatepicker)

3. Images not showing

Problem: Images broken or not loading
Solution:
  • Check image paths in public/images/ folder
  • Make sure file names match those in HTML
  • Check file permissions on server

4. Mobile menu not responsive

Problem: Mobile menu not appearing or not working
Solution:
  • Make sure navbar.js is loaded
  • Check element IDs in HTML
  • Verify Tailwind classes for responsive design

Debug Tools:

  • Browser DevTools: F12 to inspect elements and console
  • Network Tab: Check loading resources
  • Console: View JavaScript errors
  • Responsive Mode: Test on different screen sizes
‚úÖ Support: If you still have issues, check Tailwind CSS documentation or open an issue in the project repository.
